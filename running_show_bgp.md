- To view BGP neighbors under a specific `network-instance`, run `show network-instance <network-instance name> protocols bgp neighbor`.

  The output contains details such as `Peer` peer, `Group` peer group name, `Flags`, `Peer-As`, `State`, `Uptime`, `AFI/SAFI` Address Family Identifier / Subsequent Address Family Identifier, `[Rx/Active/Tx]` and `Summary`.

  ```
  A:admin@leaf1# show network-instance default protocols bgp neighbor
  --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
  BGP neighbor summary for network-instance "default"
  Flags: S static, D dynamic, L discovered by LLDP, B BFD enabled, - disabled, * slow
  --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
  --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
  +--------------------+------------------------------+--------------------+-------+-----------+-----------------+-----------------+---------------+------------------------------+
  |      Net-Inst      |             Peer             |       Group        | Flags |  Peer-AS  |      State      |     Uptime      |   AFI/SAFI    |        [Rx/Active/Tx]        |
  +====================+==============================+====================+=======+===========+=================+=================+===============+==============================+
  | default            | 172.16.1.0                   | ebgp-spines        | S     | 65999     | established     | 0d:0h:15m:34s   | evpn          | [0/0/0]                      |
  |                    |                              |                    |       |           |                 |                 | ipv4-unicast  | [1/0/0]                      |
  |                    |                              |                    |       |           |                 |                 | ipv6-unicast  | [0/0/0]                      |
  | default            | 172.16.2.0                   | ebgp-spines        | S     | 65999     | established     | 0d:0h:15m:33s   | evpn          | [0/0/0]                      |
  |                    |                              |                    |       |           |                 |                 | ipv4-unicast  | [1/0/0]                      |
  |                    |                              |                    |       |           |                 |                 | ipv6-unicast  | [0/0/0]                      |
  +--------------------+------------------------------+--------------------+-------+-----------+-----------------+-----------------+---------------+------------------------------+
  --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
  Summary:
  2 configured neighbors, 2 configured sessions are established, 0 disabled peers
  0 dynamic peers
  
  --{ running }--[  ]--
  A:admin@leaf1#
  ```
  The `[Rx/Active/Tx]` explaination:
    - `Rx`: the numbers of received prefixes (before routing-policy)
    - `Active`: the numbers of active prefixes (after routing-policy)
    - `Tx`: the numbers of advertised prefixes
